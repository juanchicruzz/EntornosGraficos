1.

HTML es el acrónimo de HyperText Markup Language, o en español, Lenguaje para el Formato de Documentos de Hipertexto. Es un lenguaje que es comprendido universalmente por todas las computadoras para representar la estructura y la interacción de una página web. El mismo permite a aquellos que crean y distribuyen HTML estructurar los documentos en línea, proveer a los usuarios interacciones como clicks en vínculos o manejar archivos de audio/video, así como utilizar tablas, listas, imágenes, hojas de cálculo o incluso crear formularios para obtener información de parte de los usuarios. Desde 1980 hasta la actualidad HTML ha sufrido varias modificaciones y ha pasado por distintas versiones, incluyendo:

  • HTML 2.0
  • HTML 3.0
  • HTML 3.2
  • HTML 4 , en el cual se avanzó sustancialmente al contar con mecanismos para hojas de estilo, ejecución de scripts, marcos, objetos incluidos, soporte mejorado para texto de derecha a izquierda y direcciones mezcladas, tablas más ricas y mejoras en formularios, ofreciendo mejoras de accesibilidad para personas con discapacidades
  • HTML 4.01 que es una versión que corrige algunos errores de la version 4
  • HTML 5 el cual es usado en la actualidad y tal vez el más importante al haber traído consigo muchas ventajas y mejoras que se explican mejor en otro apartado de la práctica

Se considera desaprobado a un elemento o atributo que quedó anticuado por el simple hecho de que existen nuevas extructuras y que por lo tanto su uso ya no es recomendado, esto debido a las actualizaciones del estándar HTML. Los elementos desaprobados pueden declararse obsoletos en versiones futuras de HTML. Un elemento o tag obsoleto es aquel para el cual no hay garantia de soporte por parte de un agente de usuario. En la especificación sólo se enumeran por razones históricas en el manual de referencia pero su definición ha sido removida. Ejemplos de estas etiquetas obsoletas por ejemplo son 'big', 'center' o 'font'.

La especificación HTML permite a los autores especificar metadatos, es decir, información adicional del documento más que sólo su contenido. Para esto utiliza el elemento META, especificando una propiedad y un valor para la misma. Por ejemplo, puede utilizarse el elemento < META name="Author" content="Joshua"> para definir el autor de un documento. La especificación de metadatos implica dos pasos:
a. La declaración de una propiedad y de un valor para la misma. Esto puede hacerse de dos maneras; a través del elemento META desde dentro del documento como se explicó aqui arriba, o bien por fuera del documento vinculando los metadatos a través del elemento LINK.
b. Luego se referencia a un perfil en el que se definen la propiedad y sus valores legales, para esto previamente se debe haber definido el atributo profile del elemento HEAD.
