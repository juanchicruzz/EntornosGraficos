Ejercicio 1

  1. ¿Qué es HTML, cuando fue creado, cuáles fueron las distintas versiones y cuál es la última?
  2. ¿Cuáles son los principios básicos que el W3C recomienda seguir para la creación de documentos con HTML?
  3. En las Especificaciones de HTML, ¿cuándo un elemento o atributo se considera desaprobado? ¿y obsoleto?>
  4. ¿Qué es el DTD y cuáles son los posibles DTDs contemplados en la especificación de HTML 4.01?
  5. ¿Qué son los metadatos y cómo se especifican en HTML?

1.

HTML es el acrónimo de HyperText Markup Language, o en español, Lenguaje para el Formato de Documentos de Hipertexto. Es un lenguaje que es comprendido universalmente por todas las computadoras para representar la estructura y la interacción de una página web. El mismo permite a aquellos que crean y distribuyen HTML estructurar los documentos en línea, proveer a los usuarios interacciones como clicks en vínculos o manejar archivos de audio/video, así como utilizar tablas, listas, imágenes, hojas de cálculo o incluso crear formularios para obtener información de parte de los usuarios. Desde 1980 hasta la actualidad HTML ha sufrido varias modificaciones y ha pasado por distintas versiones, incluyendo:

  • HTML 2.0
  • HTML 3.0
  • HTML 3.2
  • HTML 4 , en el cual se avanzó sustancialmente al contar con mecanismos para hojas de estilo, ejecución de scripts, marcos, objetos incluidos, soporte mejorado para texto de derecha a izquierda y direcciones mezcladas, tablas más ricas y mejoras en formularios, ofreciendo mejoras de accesibilidad para personas con discapacidades
  • HTML 4.01 que es una versión que corrige algunos errores de la version 4
  • HTML 5 el cual es usado en la actualidad y tal vez el más importante al haber traído consigo muchas ventajas y mejoras que se explican mejor en otro apartado de la práctica

2.

Los principios generales para la creación de documentos HTML son:

  • Separación entre estructura y presentación.
    A medida que HTML avanzó los elementos o atributos que se utilizaban para mejorar la presentación de los mismos se vieron reemplazados por las hojas de estilos. Esto facilita la revisión del documento y enfocando el código HTML para servir de estructura que es para lo que fue creado.
  • Accesibilidad universal.
    Los autores deben tener en cuenta en la representación de sus documentos para servir a una amplia gama de usuarios finales, esto incluye a aquellos que presentan dificultades físicas o cognitivas. Para esto se podrían implementar navegadores basados en voz, lector braile, o las herramientas que proporciona HTML como por ejemplo el atributo alt , el atributo acceskey, entre otros.
  • Representación incremental.
    Con las nuevas tablas introducidas en HTML4, los autores pueden ayudar a los agentes de usuario a representar los documentos más rápidamente. Los autores pueden aprender a diseñar tablas para su representación incremental.

3.

Se considera desaprobado a un elemento o atributo que quedó anticuado por el simple hecho de que existen nuevas extructuras y que por lo tanto su uso ya no es recomendado, esto debido a las actualizaciones del estándar HTML. Los elementos desaprobados pueden declararse obsoletos en versiones futuras de HTML.
Un elemento o tag obsoleto es aquel para el cual no hay garantia de soporte por parte de un agente de usuario. En la especificación sólo se enumeran por razones históricas en el manual de referencia pero su definición ha sido removida. Ejemplos de estas etiquetas obsoletas por ejemplo son 'big', 'center' o 'font'.

4.

La DTD (definición del tipo de documento) es la estructura reglamentaria, es decir, los elementos y atributos que están disponibles para cada tipo de documento. Esta debe ser la primera linea en un documento HTML, incluso antes de la etiqueta < html >. La DTD en si no es una etiqueta HTML, en cambio es una instrucción para inidcar al navegador acerca de la versión del lenguaje en que esta escrita nuestra página web.
Contrastando un documento con su DTD se puede comprobar si éste es válido o no. En la especificación de HTML 4.01 existen tres tipos de DSD:

  1. HTML 4.01 Strict: Es la forma mas estricta de las DTD. Aqui se da prioridad a la estructura antes que la presentacion, la DTD contiene todos los elementos y atributos pero no incluye elementos de presentacion y aquellos en desuso.
  2. HTML 4.01 Transitional: Incluye todos los elementos del DTD Strict pero ademas agrega cualidades presentacionales, elementos desaprobados y elementos obsoletos, aunque no incluye los marcos.
  3. HTML 4.01 Frameset: Esta DTD es igual a la Transitional pero sí incluye los marcos

5.

La especificación HTML permite a los autores especificar metadatos, es decir, información adicional del documento más que sólo su contenido. Para esto utiliza el elemento META, especificando una propiedad y un valor para la misma. Por ejemplo, puede utilizarse el elemento < META name="Author" content="Joshua"> para definir el autor de un documento. La especificación de metadatos implica dos pasos:
a. La declaración de una propiedad y de un valor para la misma. Esto puede hacerse de dos maneras: a través del elemento META desde dentro del documento como se explicó aqui arriba, o bien por fuera del documento vinculando los metadatos a través del elemento LINK.
b. Luego se referencia a un perfil en el que se definen la propiedad y sus valores legales, para esto previamente se debe haber definido el atributo profile del elemento HEAD.
